## Key AWS Services to Master

### Domain 1 (Monitoring)
- Amazon CloudWatch (metrics, alarms, dashboards, agent, Logs Insights)
- AWS CloudTrail
- Amazon EventBridge
- AWS Systems Manager (Automation runbooks)
- Amazon SNS
- Amazon Managed Service for Prometheus

### Domain 2 (Reliability)
- Auto Scaling (EC2, ECS, DynamoDB)
- Elastic Load Balancing (ALB, NLB, GLB)
- Amazon Route 53 (health checks)
- AWS Backup
- Amazon RDS (Multi-AZ, read replicas, PITR)
- Amazon S3 (versioning, replication)
- Amazon ElastiCache, CloudFront

### Domain 3 (Deployment)
- AWS CloudFormation & AWS CDK
- EC2 Image Builder
- AWS Systems Manager
- AWS Lambda
- Terraform & Git basics
- AWS RAM, CloudFormation StackSets

### Domain 4 (Security)
- AWS IAM (policies, roles, federation, MFA)
- AWS KMS & ACM
- AWS Secrets Manager / SSM Parameter Store
- AWS Security Hub, GuardDuty, Inspector, Config
- AWS Trusted Advisor
- AWS Organizations & SCPs

### Domain 5 (Networking)
- Amazon VPC (subnets, NACLs, security groups, NAT, IGW)
- Route 53 (routing policies, Resolver, DNS Firewall)
- CloudFront & Global Accelerator
- AWS WAF, Shield, Network Firewall
- VPC Flow Logs, Transit Gateway
- VPC Endpoints, PrivateLink

## Tips for Exam Day
1. Read EVERY word in the question - AWS loves tricky wording
2. Look for keywords: "MOST cost-effective", "LEAST operational overhead", "FASTEST"
3. Eliminate obviously wrong answers first
4. If unsure, pick the AWS-managed service option (less operational overhead)
5. "SysOps" perspective = you OPERATE, not design from scratch
6. Time management: ~2 min per question, flag and move on if stuck
